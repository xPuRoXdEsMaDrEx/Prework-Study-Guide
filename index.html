HTML

  • The head element contains information, or metadata, that helps the browser to render the page correctly. Metadata is data about our webpage that the browser uses to effectively serve our webpage to users. Metadata is not typically seen by users, but it contains important information aboout accessibility, search engines, and performance.
  • The body element represents the visible content shown to the user.
  • The footer element contains the author, copyright, contact, sitemap, and navigation.
  • The (section) element allows use to elegantly organize the content on our page based on the topics we have.
  • The (title) element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO).
  • The (style) element defines the CSS styles associated with a website.
  • The (link) element connects and external resource to the HTML document.
  • The (meta) element defines metadata such as the character set, description, keywords, author, and viewpoint.
  • The (script) element embeds client-side scripts in the HTML document.
  • The charset element set our character encoding.
  • The http-equiv element set an HTTP header for content.
  • The viewpoint element sets the default view for our webpage.
  • The (ul), (ol), (li) elements represent unordered lists, ordered lists, and list items.
  • The (img) element contains information about images that are dispalyed on the webpage. It also contains various attributes such as the src attribute which is the most important because it defines the location of the image. The alt attribute contains a text string that describes the appearance and functionality of the image. The alt text is also important for screen readers and other assistive technologies while also improving search engine rankings for your website.
  • The (br) element creates aline of empty space, or a line break, between two blocks of content.
  • The (a) element which stands for anchor, creates a link to the same webpage or other webpages. the href attribute points to the URL for the link such as (a href="https://nationalzoo.si.edu/animals/giant-panda")More about pandas(/a)

Git

  • GitHub Workflow procedure. First git add -A, to place our changes in the staging area. Second git commit -m, this commits all our staged changes so that they are pushed to our remote GitHub. Third git pull origin (branch namme), so that we are working with the most recent version, it is very important while working with groups. Fourth git push origin (branch name), to push up all our changes to GitHub. Fifth create a pull request on GitHub so that we may merge our feature branch in the the base branch. Afterward navigate to that feature branch and create a pull request. Lastly we compare the two branches that we want to merge. The base branch that we want to merge in to should come first. once on the pull request form, give your PR a title and a short description of the changes then click create pull request and merge pull request.
  • Git Workflow order, first add, second commit, third push up the code in a feature branch. Afterwards open a pull request in GitHub with my feature branch as the comparison branch and the default branch as the base branch. Lastly, merge the PR so that the main branch holds the code for the project.

  • When we create a new branch to add something new to our project, we call it a feature branch. That is because we are adding a new feature or functionality to our app.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git init: initialize an existing directory as a Git repository.
  • git clone [url]: retrieve and entire repository from a hosted location via URL.
  • git add -A: This adds the modifications in the current working branch to the staging area. The staging area informs Git that the changes will be added in the next commit.
  • git branch --All: This list all our branches.
  • git commit -m "message here": This must be done first before updating the version we are working on. Afterwards our Git repo is now up to date with our working directory.
  • git push origin (branch name): This pushes up all our changes we have staged to our remote GitHub account.
