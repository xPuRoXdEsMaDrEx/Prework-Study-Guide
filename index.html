Git

  • When creating an issue on GitHub one must use markdown syntax to properly layout data and allows other developers or users to understand how to use our application.
  • When we create a new branch to add something new to our project, we call it a feature branch. That is because we are adding a new feature or functionality to our app.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git init: initialize an existing directory as a Git repository.
  • git clone [url]: retrieve and entire repository from a hosted location via URL.
  • git add -A: This adds the modifications in the current working branch to the staging area. The staging area informs Git that the changes will be added in the next commit.
