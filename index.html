HTML

  • The head element contains information, or metadata, that helps the browser to render the page correctly. Metadata is data about our webpage that the browser uses to effectively serve our webpage to users. Metadata is not typically seen by users, but it contains important information about accessibility, search engines, and performance.
  • The body element represents the visible content shown to the user.
  • The footer element contains the author, copyright, contact, sitemap, and navigation.
  • The (section) element allows use to elegantly organize the content on our page based on the topics we have.
  • The (title) element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO).
  • The (style) element defines the CSS styles associated with a website.
  • The (link) element connects and external resource to the HTML document.
  • The (meta) element defines metadata such as the character set, description, keywords, author, and viewpoint.
  • The (script) element embeds client-side scripts in the HTML document.
  • The charset element set our character encoding.
  • The http-equiv element set an HTTP header for content.
  • The viewpoint element sets the default view for our webpage.
  • The (ul), (ol), (li) elements represent unordered lists, ordered lists, and list items.
  • The (img) element contains information about images that are dispalyed on the webpage. It also contains various attributes such as the src attribute which is the most important because it defines the location of the image. The alt attribute contains a text string that describes the appearance and functionality of the image. The alt text is also important for screen readers and other assistive technologies while also improving search engine rankings for your website.
  • The (br) element creates aline of empty space, or a line break, between two blocks of content.
  • The (a) element which stands for anchor, creates a link to the same webpage or other webpages. the href attribute points to the URL for the link such as (a href="https://nationalzoo.si.edu/animals/giant-panda")More about pandas(/a)

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside the element.
  • CSS Rule Ex div {color: blue; text-align: justify;} {Selector: Declaration Block;}
  • A selector defines the element or attributes to which the rule is applied
  • Declarations contain two important components: the CSS property we want to apply and the value for that property, blue.
  • The margin property acts much like a margin in a word document. It indicates how much space we want around outside of an element. The margin is not visible on the page; it merly adds a buffer. When we set the margin property to 0, we indicate that we want NO extra space around the element.
  • The padding property adds space around the content inside an element. When you increase padding, you are making the entire element bigger. That is because the padding, while transparent, is located inside the element, whereas the margin is the transparent buffer outside an element. Not the same as margin.
  • Use the * selector when we want to apply a rule to all the elements visible on the page.
  • We can combine two or more electors into a single CSS rule with a comma, allowing the declarations to affect both selectors. This helps us write DO NOT REPEAT YOURSELF (DRY) code.
  • The img selector display declaration value block, assigns each image to start on a new line.
  • In order to see our style changes reflect onto our HTML code we must use a link element which creates the connection to the external file. The following code should be placed after the final meta element and before the title within the head section. This is where we store metadata, which contains information about the webpage.
  • ^link rel="stylesheet" href="./assets/style.css"^
  • The link attribute specifies the relationship between the currentdocument and the linked document resource.
  • The href attribute specifies the location (URL) of the external source. The href attribute is using a relative path. This means the location is derived from the current location (in this case, index.html)

Git

  • GitHub Workflow procedure. First git add -A, to place our changes in the staging area. Second git commit -m, this commits all our staged changes so that they are commited to our remote GitHub. Third git pull origin (branch namme), so that we are working with the most recent version, it is very important while working with groups. Fourth git push origin (branch name), to push up all our changes to GitHub. Fifth create a pull request on GitHub so that we may merge our feature branch in the the base branch. Afterward navigate to that feature branch and create a pull request. Lastly we compare the two branches that we want to merge. The base branch that we want to merge in to should come first. once on the pull request form, give your PR a title and a short description of the changes then click create pull request and merge pull request.
  • Git Workflow order, first add, second commit, third push up the code in a feature branch. Afterwards open a pull request in GitHub with my feature branch as the comparison branch and the default branch as the base branch. Lastly, merge the PR so that the main branch holds the code for the project.
